# Release instructions

- Keep `CHANGELOG.md` as the release notes source for GitHub releases.
- Every release note entry must include simple copyable install lines for Linux, Windows, and macOS.
- Update the top changelog entry before tagging a release so the generated release body includes the install commands.
- Use the current release version in all three install commands and in `README.md`.
- Verify the release body is copyable and platform-specific before publishing.
